Busy Light Help

Troubleshooting

Cannot connect to Home Assistant

  • Make sure your Home Assistant URL is correct and includes the port number if needed (e.g. http://homeassistant.local:8123).
  • Verify that your Mac can reach the Home Assistant server. Try opening the URL in a web browser.
  • If you use HTTPS, ensure your certificate is valid or trusted by your Mac.
  • Check that your firewall or network settings allow connections on the Home Assistant port.

Token not recognized or authentication errors

  • Make sure you copied the entire token with no extra spaces or line breaks.
  • Long-lived access tokens can expire if they are revoked in Home Assistant. Create a new token and paste it into the Token field.
  • Verify the token belongs to a user with permission to activate scenes.
Your token is stored in the macOS Keychain. If you are prompted to unlock the Keychain, allow access so Busy Light can retrieve the token on launch.

Scenes not appearing in the list

  • Scenes must be configured in Home Assistant before they appear in Busy Light. Check your HA dashboard to confirm they exist.
  • The scene list is fetched when you open the Scenes tab. Make sure your connection is working first (a green checkmark on the Home Assistant tab).
  • Only entities whose ID starts with scene. are shown.

Triggers not firing

  • Open Settings > Triggers and confirm that the trigger toggle is turned on and a scene is selected.
  • The webcam trigger takes priority over the microphone trigger. If both fire at the same time, only the webcam scene is used.
  • Focus Mode detection is experimental and reads from a system file that may change between macOS versions. If it stops working after a macOS update, it will degrade gracefully.

Camera or microphone not detected

Busy Light does not need camera or microphone permissions. It reads device metadata (whether the device is in use) rather than accessing the actual audio or video stream.

  • Some virtual cameras (e.g. software-only camera drivers) may not report their status through the system APIs that Busy Light monitors.
  • If detection stops working, try quitting and relaunching Busy Light.

Resetting the app

To reset Busy Light to its first-run state, open Terminal and run:

defaults delete com.mboszko.BusyLight

This removes all preferences including scenes, triggers, and display settings. Your Home Assistant token is stored separately in the Keychain and is not affected by this command.
